# Sky Jump — Image Asset Spec

만두 미니게임 "Sky Jump" 에 필요한 이미지 자산 명세서.
구현은 로직 우선 — 실제 이미지가 들어오기 전까지는 색 네모 placeholder 로 동작.

게임 뷰포트: **480 x 480** (메인 창과 동일 크기).
모든 자산은 **PNG (투명 배경)** 권장. 픽셀 단위 크기는 1x (Retina 대응이 필요하면 2x 별도 파일로).

---

## 캐릭터 (만두)

| 파일명 | 권장 크기 (px) | 설명 |
| --- | --- | --- |
| `mandu.png` | 80 x 80 | 만두 기본 이미지 — 점프/하강/좌우 이동 모두 이 한 장으로 처리 |

> 만두 hitbox 는 코드 상 60 x 60 정도로 잡고, 이미지는 80 x 80 으로 여유 padding 을 둠. 별도 jump/fall 분리 없이 단일 이미지로 통일.

## 배경 / 환경

| 파일명 | 권장 크기 (px) | 설명 |
| --- | --- | --- |
| `bg_sky.png`     | 480 x 480 | 게임 배경. 연한 하늘색 그라데이션 + 옅은 구름 텍스처 |
| `bg_sky_tile.png`| 480 x 480 | 위로 스크롤되었을 때 끝없이 이어지도록 vertically tileable |
| `ground.png`     | 480 x 80  | 게임 시작 시 화면 하단에 깔리는 땅. 풀밭+흙 |

> 배경은 단일 이미지로 충분. 스크롤 시 시차(parallax) 효과 주려면 `bg_sky_far.png`, `bg_sky_near.png` 로 분리 가능 (옵션).

## 색상 placeholder 매핑 (이미지 들어오기 전까지)

구현 단계에서는 실제 이미지 대신 다음 색의 네모 사각형으로 렌더한다:

| 자산 | placeholder 색 |
| --- | --- |
| 만두 | `#F0C8A0` (만두 톤 베이지) + 보더 `#5C3D2E` |
| 구름 플랫폼 | `#FFFFFF` + 보더 `#B0D0E0` |
| 땅 | `#A8D080` (풀밭) + 하단 `#8B6A3F` (흙) |
| 배경 하늘 | linear-gradient `#CDE7FF → #E8F4FF` |

이 매핑으로 모든 로직(충돌, 점프, 스크롤, 점수)이 정상 동작하는지 검증한 뒤, 실제 이미지가 들어오면 placeholder 렌더 함수만 이미지 draw 로 교체하면 된다.
