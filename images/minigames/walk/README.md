# 산책 미니게임 이미지 자산

산책 게임 풀밭/만두 자산을 여기에 넣어주세요.
자산이 없어도 게임은 동작하지만 (단색 풀밭 + `mandu_up.png` 폴백), 이미지를 채우면 진짜 산책 느낌이 살아납니다.

## 파일 목록

### `grass_tile.png` — 풀밭 타일 (권장)

게임 배경 전체를 덮는 **seamless 타일**. 위→아래로 끊김 없이 스크롤됩니다.

- **권장 크기**: 정사각 **256x256 px** (게임 코드에서 ROAD_TILE = 256 으로 가정).
  - 다른 사이즈를 쓰고 싶으면 `minigames/walk/walk.js` 의 `ROAD_TILE` 도 같이 바꾸면 됩니다.
- **포맷**: 일반 PNG (투명 X — 풀밭이 화면을 가득 덮어야 함).
- **Seamless**: 좌우/상하 가장자리가 자연스럽게 이어져야 합니다 (이음매가 보이지 않도록).
- **분위기**: 너무 화려하지 않게. 풀잎이 살짝 보이고, 톤 차이로 풀밭처럼 보이는 정도. 작은 꽃 한두 송이/돌멩이 정도 변화는 OK.
- **색감**: 게임 wrap 배경(`#D9EFB0` ~ `#B8DD8C` 그라데이션)과 어울리는 초록 톤 권장.
- 자산이 없으면 단색 초록 그라데이션만 보입니다 (게임은 정상 동작).

### `mandu_top.png` — 필수 권장

### `mandu_top.png` — 필수 권장

- **시점**: 위에서 내려다본 만두 (탑다운 / 도그캠).
- **방향**: 머리가 화면 위쪽을 향하도록 그려주세요. (게임에서 만두가 위로 걸어가는 것처럼 보여야 합니다.)
- **권장 크기**: 256x256 px 이상 정사각형. (게임에서 96x96 으로 렌더되지만 레티나 대비 2x 권장.)
- **포맷**: 투명 PNG.
- **빛/그림자**: 위에서 내려쬐는 부드러운 그림자 정도면 충분. 너무 짙은 캐스트 섀도우는 피해주세요 (게임에서 별도 drop-shadow 가 들어갑니다).

### `mandu_top_walk_a.png`, `mandu_top_walk_b.png` — 선택

걷는 모션을 더 살리고 싶을 때 2장의 프레임을 추가로 제공해주세요.
- `_walk_a`: 앞다리 왼쪽이 앞으로 나간 자세.
- `_walk_b`: 앞다리 오른쪽이 앞으로 나간 자세.
- 위와 동일한 크기/포맷/방향.
- 두 파일이 모두 존재하면 게임 중 약 220ms 마다 두 프레임을 교대로 표시해서 자연스러운 걷기 애니메이션이 됩니다.
- 둘 중 하나라도 없으면 자동으로 `mandu_top.png` 한 장만 사용합니다.

## 로딩 폴백 순서

1. `images/minigames/walk/mandu_top.png` (이 폴더)
2. `images/mandu_up.png` (프로젝트 기본 만두)
3. 색 사각형 placeholder (둘 다 실패한 경우)

## 만들 때 팁

- 도그캠 시점이라 만두의 머리/등이 잘 보여야 합니다. 얼굴은 안 보여도 OK — 뒤통수나 위에서 본 등판이 자연스럽습니다.
- 길 배경이 노란-갈색 톤(`#FFE8C8` ~ `#FFC270`)이라, 그 위에서 잘 떨어져 보이는 외곽선/색조를 권장합니다.
- 그림자는 만두 아래에 옅게(20~30% 불투명) 깔린 타원 정도면 충분합니다 — 게임에서 추가 drop-shadow 가 적용됩니다.
